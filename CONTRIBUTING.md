# Contributing to ROE Gate

Thank you for your interest in ROE Gate.

## Development Model

ROE Gate is developed entirely in-house by [Grey Line Interactive](https://roegate.io). **We do not accept pull requests or external code contributions.** All PRs from external contributors will be closed without review.

This is a deliberate decision — ROE Gate is a security-critical tool with patent-pending architecture, and all code is authored, reviewed, and tested internally.

## How You Can Help

We welcome your involvement in other ways:

### Report Bugs
If you find a bug, please [open an issue](https://github.com/Grey-Line-Interactive/ROEGATE/issues/new?template=bug_report.yml) with steps to reproduce it.

### Suggest Features
Have an idea for an improvement? [Open a feature request](https://github.com/Grey-Line-Interactive/ROEGATE/issues/new?template=feature_request.yml) and describe your use case.

### Join the Discussion
Ask questions, share how you're using ROE Gate, and discuss ideas in [GitHub Discussions](https://github.com/Grey-Line-Interactive/ROEGATE/discussions).

### Report Security Vulnerabilities
If you discover a security vulnerability, **do not open a public issue.** See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.
